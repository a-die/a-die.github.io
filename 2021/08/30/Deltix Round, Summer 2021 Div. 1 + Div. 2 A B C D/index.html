A. A Variety of Operations

题目链接

题意：
给定两个数$a,b$，初始值为$0$，有三个操作，$a$和$b$都加$k$；或者$a$加$k$,$b$减$k$；或者$b$加$k$,$a$减$k$。问最少执行多少次操作，使得$a=c,b=d$。无法完成则输出$-1$。

分析：
因为在任意变化之后，$a-b$的奇偶性是不会发生变化的，因为初始$a,b$都是$0$，$a-b=0$。所以如果$c-d$的差是奇数，则不可能得到这个数对。
而如果$c-d$的差是偶数，又要分几种情况讨论

  • 如果$c=0$且$d=0$，那么不需要操作，输出$0$。
  • 如果$c=d$且不等于$0$，那么需要操作$1$次
  • 其他情况需要操作$2$次，我们可以使用第二种类型或第三种类型的运算$k=\frac {|c−d|}{2}$。如果$c>d$则使用第二种，$c<d$使用第三种。在此之后，假设$c>d$，再使用第一种类型的操作，$k=c-\frac {|c−d|}{2}$。$c<d$同理。

代码： 

#include <iostream>
#include <cstdio>
#include <algorithm>
#include <cstring>
#include <cmath>
using namespace std;

int main()
{
    int t;scanf("%d",&t);
    while(t--){
        int c,d;scanf("%d %d",&c,&d);
        if(c==0&&d==0)printf("0\n");
        else if((c-d)%2)printf("-1\n");
        else if(c==d)printf("1\n");
        else printf("2\n");
    }
    return 0;
}

B. Take Your Places!

题目链接

题意：
给定一个数组$a$，每一次可以交换相邻的两个元素，问最少交换多少次使得整个元素每两个相邻元素之间的奇偶性不同，也就是整个数组是奇偶交替的。

分析：
贪心，贪心思路对了就很好想了。首先，判断无法成立很容易，如果是偶数个元素，判断是否奇偶数各占一半，如果是奇数个元素，判断是否奇数比偶数多$1$或者少$1$。如果无法成立，则输出$-1$。
否则，我们只需要考虑两种情况

  • 第一个元素是奇数。
  • 第一个元素是偶数。

也就是说，我们最后的数组一定是$10101$，或者$01010$这样的($1$看成奇数，$0$看成偶数)。我们可以将当前数组的奇数位置存储下来，如果数组总个数$n$为奇数，那么如果奇数多就将奇数放在奇数位置，奇数少就将奇数放在偶数位置，依次放入，暴力求移动次数。如果数组总个数$n$为偶数，那么奇数放在奇数位置算一次，奇数放在偶数位置也算一次移动次数，取$min$即可。

代码： 

#include <iostream>
#include <cstdio>
#include <algorithm>
#include <cstring>
#include <cmath>
#include <vector>
#define ll long long
using namespace std;
const int N=1e5+5;
int a[N],temp[N];
int main()
{
    int t;scanf("%d",&t);
    while(t--){
        int n;scanf("%d",&n);
        vector<int>odd;
        for(int i=1;i<=n;i++){
            scanf("%d",&a[i]);
            if(a[i]%2)odd.push_back(i);
        }
        if((odd.size()*2!=n&&n%2==0)||(odd.size()*2!=n-1&&odd.size()*2!=n+1&&n%2==1)){
            puts("-1");
            continue;
        }
        int k=1,res=0,cnt=0;
        if(n%2==0){
            for(auto x:odd){
                res+=abs(x-k);k+=2;
            }
            k=2;
            for(auto x:odd){
                cnt+=abs(x-k);k+=2;
            }
        }
        else{
            if(odd.size()*2==n-1)k=2;
            else k=1;
            for(auto x:odd){
                res+=abs(x-k);k+=2;
            }cnt=res;
        }
        printf("%d\n",min(res,cnt));
    }
    return 0;
}

C. Compressed Bracket Sequence

题目链接

题意：
给定一个括号序列，这个括号序列是给的每一次连续的括号的数量，第$i$个位置，如果$i$是奇数，则给定的是$’(‘$的括号数量，如果i是偶数，则给定的是$’)’$的括号数量。例如，对于括号序列$“(())())”$，相应的数字序列是$[3,2,1,3]$。求有多少个匹配的括号子片段。

分析：
说在前面，这一段都比较抽象，可以自己在纸上模拟一下过程。感觉描述不是很到位，可以边看代码边模拟手算理解
用$sum$维护可能对后续有影响的左括号数量。
分成奇偶讨论，如果是奇数位置，当前是左括号，直接加上左括号的数量$c$。
如果当前是偶数位置，则是右括号的数量$c$，用$res$维护最后的结果，当前$res$肯定要加上$min(sum,c)$。
然后$sum$减去右括号的数量$c(sum-=c)$，这个时候我们需要判断当前的匹配括号在后面的匹配括号中，是否能够形成新的子片段。
如果此时$sum<0$。说明在当前位置的右括号数量比左侧的左括号数量还多，那么当前包括左侧括号一定在后续的过程中无法构成新的括号子片段了，将$sum$清$0$，当前包括左侧括号全部清空，对后续操作没有影响了。
我们将当前$sum$存储在$s$数组中，因为在后面的匹配了的括号中，可能还会形成新的括号子片段。所以当前的$sum$还需要判断是否能够和之前匹配的括号中，形成新的匹配子片段。我们遍历之前存储的$s$数组

  • 如果$sum=s[i]$，说明在$s[i]$的位置到当前的位置中，就是一个匹配的括号段，所以左括号的数量都没有发生变化，所以可以多一个匹配子片段
  • 如果$sum<s[i]$，说明在当前位置出现了多的右括号，抵消了相对于$s[i]$来说的部分左括号。那么我们可以在$s[i]$中取这被抵消的左括号出来形成一个新的括号段。
    但是对于再后面的括号来说，这里的$s[i]$被抵消了部分，相当于这里的$s[i]$抵消的部分和当前匹配的串捆绑在一起了，后续的括号也是和剩下的$s[i]$的括号匹配，那我们索性将$s[i]$直接置为无穷小，再存入抵消之后剩下的括号数量。
  • 如果$sum>s[i]$，对于$s[i]$中剩下的左括号，都找不到当前位置的右括号，所以不能形成新的括号段。

代码： 

#include <iostream>
#include <cstdio>
#define ll long long
using namespace std;
const int N=10005;
ll s[N];
int main()
{
    int n;
    scanf("%d",&n);
    ll sum=0,res=0,idx=0;
    for(int i=1;i<=n;i++){
        ll c;scanf("%lld",&c);
        if(i%2)sum+=c;
        else{
            res+=min(sum,c),sum-=c;
            for(int j=0;j<idx;j++){
                if(s[j]==sum)res++;
                else if(s[j]>sum)s[j]=-1e18,res++;//前面留下来的在后面再也用不到了
            }
            s[idx++]=sum;
            if(sum<0)sum=0,idx=0;
        }
    }
    printf("%lld\n",res);
    return 0;
}

D. Take a Guess

题目链接

题意：
交互题，对于一个序列$a$，你可以询问不超过$2*n$个问题，两个下标为$i$和$j$的项按位与的结果是什么$(i≠j)$，两个下标为i和j的项按位或$(i≠j)$的结果是什么?你需要找到第$k$小数。

分析：
感觉这题是第二简单的题，只要知道$a+b=a&b+a|b$，就可以完成了。(记得牛客好像做过这个定理的题目)。
每次询问相邻两个数的与和或的值，记录一下相邻两个数的和，最后再计算一下第一个和第三个数字的和，就可以通过三个式子($a_{1}$和$a_{2}$的和，$a_{2}$和$a_{3}$的和，$a_{1}$和$a_{3}$的和)先求得$a_{1}$的值，我们又知道所有相邻两项的和，其他项自然就都能得到了，最后从小到大排序，输出第k小的数字。

代码： 

#include <iostream>
#include <cstdio>
#include <cstring>
#include <algorithm>
using namespace std;
const int N=3e4+5;
int b[N],a[N];
int main()
{
    int n,k;
    scanf("%d %d",&n,&k);
    for(int i=1;i<=n-1;i++){
        printf("and %d %d\n",i,i+1);
        fflush(stdout);
        int x;scanf("%d",&x);
        printf("or %d %d\n",i,i+1);
        fflush(stdout);
        int y;scanf("%d",&y);
        b[i]=x+y;
    }
    printf("and %d %d\n",1,3);
    fflush(stdout);
    int x;scanf("%d",&x);
    printf("or %d %d\n",1,3);
    fflush(stdout);
    int y;scanf("%d",&y);
    a[1]=(x+y-b[2]+b[1])/2;
    for(int i=1;i<=n-1;i++){
        a[i+1]=b[i]-a[i];
    }
    sort(a+1,a+1+n);
    printf("finish %d\n",a[k]);
    return 0;
}
